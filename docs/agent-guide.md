## Schema Plan

### `dailySummary`

Use a singleton-style document for the homepage content.

Suggested fields:

- `careRecipientName` ‚Äî string
- `relationship` ‚Äî string
- `photo` ‚Äî image
- `moodStatus` ‚Äî string with options:
  - Calm
  - Restless
  - Engaged
  - Sleeping
- `shortSummary` ‚Äî text
- `notesOrFlags` ‚Äî array of strings or text
- `activities` ‚Äî array of objects:
  - `title` ‚Äî string
  - `time` ‚Äî string
  - `note` ‚Äî optional string

### `blogPost`

Suggested fields:

- `title` ‚Äî string
- `slug` ‚Äî slug from title
- `author` ‚Äî string
- `publishedAt` ‚Äî datetime/date
- `featuredImage` ‚Äî image
- `excerpt` ‚Äî text
- `body` ‚Äî portable text / block content

### `resourceCategory`

Suggested fields:

- `title` ‚Äî string
- `slug` ‚Äî slug
- `description` ‚Äî text

Suggested categories:

- Medical
- Legal & Financial
- Daily Living
- Emotional Support

### `resource`

Suggested fields:

- `title` ‚Äî string
- `shortDescription` ‚Äî text
- `thumbnail` or `icon` ‚Äî image
- `externalUrl` ‚Äî url
- `category` ‚Äî reference to `resourceCategory`

## Data Modeling Guidance

Use simple, editor-friendly structures.

Preferred modeling decisions:

- `dailySummary` should behave like a single editable document
- `resource` should reference `resourceCategory`
- `blogPost` should use a proper slug and portable text body

Avoid:

- unnecessary relational complexity
- too many document types
- custom studio features unless required

---

## UI Guidance

Design tone:

- calm
- readable
- supportive
- clean
- lightly polished

Use:

- card-based layouts
- consistent spacing
- subtle shadow/radius
- restrained color palette
- obvious section headings
- strong mobile readability

Avoid:

- flashy gradients
- over-animated UI
- overly clinical or enterprise-heavy visuals

### Suggested blog structure

- responsive card grid
- image

### Suggested resources structure

- responsive resource cards
